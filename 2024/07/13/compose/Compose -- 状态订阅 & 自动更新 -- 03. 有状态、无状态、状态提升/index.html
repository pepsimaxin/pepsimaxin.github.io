有状态、无状态、状态提升

Source Code based on: androidx.compose.**:**:1.5.0

先来看两段 Compose 的官方描述：

  1. 利用 Compose，您可以构建不与特定 activity 或 fragment 相关联的小型「无状态」组件，这让您可以轻松重用和测试这些组件。

  2. 使用 remember 存储对象的可组合项包含「内部状态」，这会使该可组合项「有状态」。在调用方不需要控制状态，并且不必自行管理状态便可使用状态的情况下，「有状态」会非常有用。但是，具有「内部状态」的可组合项往往不易重复使用，也更难测试。

所以我们可以抛出三个问题：

  1. 什么是「无状态」？
  2. 什么是「有状态」？
  3. 什么是「内部状态」？

带着这三个问题，我们开始探讨 Compose 中的状态。

无状态
